Task: create a new Drizzle schema change — $ARGUMENTS

1. Read `docs/DATA-MODEL.md` and the DB section of `docs/ARCHITECTURE.md` first. Don't guess at conventions.
2. Edit or create the schema file under `packages/db/src/schema/<domain>/<table>.ts`.
   - Tenant-scoped tables: `tenant_id uuid not null` is required.
   - Include `created_at` and `updated_at`, or `created_at` alone for append-only tables.
   - Add indexes driven by known read paths. Ask about query shape if it's not obvious; do not speculate.
3. Update the barrel export in `packages/db/src/schema/index.ts`.
4. Run `bunx drizzle-kit generate` and read the generated SQL line by line before moving on.
5. Triggers, CHECK constraints, and RLS policies are not generated by Drizzle. Author them as raw SQL in a separate migration file alongside the generated one.
6. Tests (under `packages/db/test/`): at minimum insert + query, and for tenant-scoped tables a cross-tenant leakage test that asserts RLS blocks the other tenant.
7. Stop and report the files changed and the generated SQL before running the migration against any database.
